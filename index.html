What emerges isn't a technology or access problem — it's an epistemic trust problem. Teams already use AI daily across all four countries, but they don't trust it for official documents. Costa Rica named Claude as the benchmark spontaneously. The question is no longer whether they'll use AI — it's which one, for what, and who governs it.

01

Universal signals — what crosses borders

Trust · Foundational

Traceability is a precondition, not a preference

This isn't "nice to have" — it's the entry condition. As one participant put it: we can't ship anything without traceability. The majority across every mission chose slower-but-cited over fast-but-unverified, and cross-check before using any AI output in an official document. "La respuesta muestra la fuente de manera que se puede validar" — CR participant.

Tools & profiles

Two profiles emerging — and Claude rising among one

Copilot leads everywhere, but two distinct user profiles are appearing. The transactional profile wants a quick reliable answer. The expert profile — economists, sector leads with a concrete need to look outward more than inward — invests time and interest, and Claude is rising here: used to draft PCRs, to bring in external evidence, and increasingly to build their own agents. The platform must serve both, not average them.

Adoption

The KP should travel with you — not be one more link

Nobody wants the KP as their main tool: they search in the platform, then refine outside it. The real ask isn't another browser tab — it's knowledge embedded into the systems they already use. This reframes integrations as the core roadmap: the KP goes where the work is, already on their machine. "Ya lo hace" [It already does it] — one CR participant.

Source quality

Complete and findable — exhaustiveness as an edge

Beyond reliable sources, the demand is exhaustiveness: the confidence that a query returns everything relevant, not a sample. This could become a genuine competitive advantage no generalist tool offers. Complete-and-findable arrives with the MCP layer, where the KP can serve its full corpus to any tool the user already works in.

Reliability · Foundational

Technical trust is the foundation everything sits on

As adoption grows, technical failures become the #1 trust risk. Costa Rica (the most mature adopter) and Panamá both reported Convergencia sync issues and a download bug — a project that doesn't appear undoes months of adoption. Reliability isn't a feature; it's the floor the whole value proposition stands on.

Emerging signal · Beyond the survey data

Users aren't just consumers — they're building their own agents

A pattern surfacing across field research and internal interviews: power users have moved beyond using AI tools to building their own custom agents by division. IFD is building a Copilot agent for state capacity alignment with curated lessons, with shared workspaces for 30–40 people. Sector leads in climate, gender and rule of law are building their own. The emerging demand: the KP as a catalog where those agents live and can be discovered — an agent repository, not just a document repository. The benchmark is no longer "be more like Claude." It's "be the place where our Claude-level agents live."

WHAT IS TRULY UNIQUE TO EACH COUNTRY

Several needs are shared across missions and belong to the universal patterns above — TdRs and Pliegos (Colombia + Panamá), PCR automation (Brasil + Panamá), lessons learned and TC knowledge products (Colombia + Brasil). What follows is only what genuinely distinguishes each country.

🇨🇴 Colombia

Draft POD as #1 opportunity. Colombia is the only mission that ranks document generation above search — already trusts the platform enough to want outputs, not just inputs.

Interoperability as a governance problem. 1,500+ agents operating without articulation; duplicated dashboards and parallel bank initiatives. Named here more sharply than anywhere else.

🇧🇷 Brasil

Portfolio intelligence demand. Disbursement projections and risks by sector/region, built manually in Excel today. The most data-engineering-shaped ask of the four.

Subnational / state-level visibility. Given Brasil's scale, lessons must be organized by state, municipality and execution mechanism — a granularity no other mission required.

🇨🇷 Costa Rica

"Be more like Claude." The only mission where a user named Claude spontaneously as the retention benchmark. Win on specialization, not by restricting generic AI.

Tool integration as the #1 switching trigger. The only country where integration with their existing stack beats cited sources as the top need — the most workflow-mature profile.

🇵🇦 Panamá

Tools built for HQ, not for executing agencies. The hottest challenge in the whole corpus (8 reports) — and unique to Panamá: the platform should reach the agencies that run projects on the ground.

Cross-window collaboration depends on one coordinator. BID Invest synergies are ad hoc, with no structural incentive. The sharpest articulation of the governance blocker.

03

Where to start — three priorities by horizon

The structural tension underneath all four missions

The platform indexes approved, published knowledge — but the living operational knowledge of a project's 6–7 year execution has nowhere to live.

It surfaces in all four countries: Panamá ("a project's history has nowhere to live", consultant deliverables vanish), Brasil (institutional memory dispersed by state/mechanism, PCR automation), Costa Rica (intermediate planning docs unsystematized — only final versions), Colombia (the Knowledge Loop — knowledge is generated but doesn't circulate). The platform ingests final approved documents; kick-off workshop minutes, mid-execution adjustments, in-progress TCs and semester reports circulate through personal drives and email — never indexed.

→ Line of action

This is a governance challenge, not a feature — like priority 3. There's already partial consensus on some document types that should be ingested (e.g. the semi-annual report), but no shared narrative yet on which documents the user actually needs to consume — minutes being the clearest open case. It requires a document classification protocol with Data Management (consistent naming, taxonomy, ingestion rules for non-final documents) before the platform can technically capture execution. Start by defining which execution-phase document types to ingest first (PMRs, amendments, kick-off minutes) — and note that capturing the execution story is also the prerequisite for Brasil's PCR automation. Frame this as a parallel track to the three priorities, owned jointly with Data Management.

Design hypothesis · Transversal

The Knowledge Platform doesn't compete in the generative AI market. It competes in the market of institutional trust. Its differential advantage isn't response speed but knowledge traceability — and that's exactly what no generalist tool can offer.

Colombia, Brasil, and Costa Rica — with distinct operational profiles and adoption curves — converge on the same signal: we don't need more information, we need information we can trust, embedded in how we already work. That is the platform the IDB has the unique opportunity to build.
